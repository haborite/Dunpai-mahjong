# 盾麻雀 和了演出仕様

## 1. 目的

和了を単なる結果ダイアログではなく、次の4点を順番に理解できる演出にする。

1. 誰が、ロン・ツモ・槍槓のどれで和了したか
2. どのような牌姿で和了したか
3. どの役が成立し、何ハンになったか
4. 盾牌防御を含め、誰から誰へ何点移動したか

演出の方向性は、雀魂の「和了した瞬間の強い節目」と、天鳳の「牌姿・役・点数を素早く正確に読める結果表示」を組み合わせる。

## 2. 参考UIから採用する要素

### 雀魂から採用

- ロン・ツモ宣言を卓上全体に大きく表示し、通常進行との境界を明確にする
- 和了時に短い光、拡大、暗転を使い、視線を和了者へ誘導する
- 高打点ほど色と効果音を強くする
- 点数移動を独立した画面または段階として見せる

派手なキャラクター演出や長い全画面エフェクトは採用しない。盾麻雀では牌姿と盾牌判定の可読性を優先する。

### 天鳳から採用

- 和了者の手牌、副露、和了牌を一列で明確に開示する
- 役名、ハン数、合計点を同じ画面で確認できる
- 情報をコンパクトにし、演出終了後は静止画として読み返せる
- クリックで演出を早送りできる

### 盾麻雀固有

- ツモ和了時は、役表示と点数移動の間に「盾牌判定」フェーズを必ず設ける
- 防御対象になった公開盾牌と、その所有者を卓上で強調する
- オープン立直によって盾牌防御が無効になった場合も明示する
- 防御成立者の支払いが0点であることを、点数移動表示でも再確認できるようにする

## 3. 標準タイムライン

自動再生時の標準時間は、役数に応じて約8秒から14秒とする。

| 時刻 | フェーズ | 表示 |
| --- | --- | --- |
| 0.0-0.5秒 | 進行停止 | 操作UIを消し、卓を少し暗くする。和了牌を発光させる |
| 0.5-1.5秒 | 和了宣言 | `ロン`、`ツモ`、`槍槓`を大きく表示。和了者名も表示する |
| 1.5-3.0秒 | 牌姿開示 | 和了者の全手牌・副露・和了牌を全員に公開する |
| 3.0秒以降 | 役表示 | 役を1件ずつ追加表示する |
| 役表示後 | 盾牌判定 | ツモ時のみ実施。防御成立または無効理由を表示する |
| 続いて | 得点確定 | 合計ハン、和了点、今回の点数移動を同時に表示する |
| 最後 | 最終結果 | 順位が確定する最終ゲームのみ、各家の累計得点と順位を表示する |
| 完了後 | 結果保持 | 全情報を静止表示し、`次局へ`を有効にする |

### 操作

- 演出中のクリックまたはSpaceキー: 現在のアニメーションを即時完了
- 再度クリックまたはSpaceキー: 次フェーズへ進む
- 全フェーズ完了後: `次局へ`ボタンを押せる
- 自動進行は設定で有効・無効を切り替えられる
- `prefers-reduced-motion`では移動・点滅を省き、フェードのみ使用する

## 4. 牌姿開示

### 表示内容

- 和了者名、席
- 門前部分の手牌
- 副露
- 和了牌
- ドラ表示牌
- ロン時は放銃者名

和了牌は通常手牌から8pxから12px離し、金色の枠と短いパルスで区別する。ロン時は河の該当牌から和了牌表示へ短い軌跡を出す。

他家が和了した場合も同じ牌姿を全プレイヤーへ送信し、必ず公開する。和了者以外の未公開手牌は開示しない。

## 5. 役表示

### 表示順

役を同時に出さず、次の順に1件ずつ表示する。

1. 状況役: 立直、オープン立直、門前清自摸和など
2. 手牌構成の役: 断么九、平和、対々和など
3. 複合・高打点役
4. ドラ
5. 赤ドラ

通常は1件あたり450msで追加する。役が7件以上ある場合、7件目以降は250msへ短縮する。各行は左から入り、役名の後にハン数を表示する。

最終行の表示後に合計ハンをカウントアップし、現在の簡略点数表に従う和了点を表示する。

## 6. 盾牌判定

盾牌判定はツモ和了時だけ表示する。

### 防御成立

1. 和了者の待ち牌一覧を小さく表示する
2. 該当する公開盾牌を卓上で青白く発光させる
3. 和了牌から該当盾牌へ細い光の線を出す
4. 中央に `盾牌防御` と対象プレイヤー名を表示する
5. `支払い 0`を表示する

複数人が防御した場合は同時に盾牌を強調し、名前と支払い0を人数分列挙する。

### オープン立直による無効

公開盾牌に待ち牌が存在していても和了者がオープン立直の場合、該当盾牌を一度青く光らせた後、赤い斜線を重ねる。

表示文言:

`オープン立直: 盾牌防御無効`

これにより「判定されなかった」のではなく、「一致したが無効化された」と分かるようにする。

### 防御なし

該当者がいない場合は専用演出を長く出さない。中央下部に `盾牌防御なし`を500ms表示して得点確定へ進む。

## 7. 点数移動

通常局でも、今回の局で発生した点数移動は和了点と同時に表示する。ただし各家の累計得点状況をまとめた一覧画面は表示しない。内部の得点は更新し、次局の中央点数表示へ反映する。

和了点欄には次をまとめて表示する。

- 合計ハンと本来の和了点
- 和了者の実獲得点を緑の`+N`
- 支払者を赤の`-N`
- 盾牌防御者を青の`防御 0`

盾牌防御によってツモ和了者の収入が減った場合、本来の和了点と実獲得点の双方を表示する。

順位が確定する最終ゲームの場合のみ、この表示の後に各家の累計得点と順位を一覧表示する。

結果パネルには最終的に次を残す。

- 和了者の牌姿
- 全役とハン数
- 合計ハン、和了点
- 各家の点数増減
- 防御成立者または防御無効理由

## 8. 打点別の演出強度

| 合計ハン | 演出 |
| --- | --- |
| 1-3ハン | 白色のフラッシュ、通常効果音 |
| 4-5ハン | 金色の縁取り、少し強い衝撃 |
| 6-7ハン | 金色の光、画面を短く揺らす |
| 8ハン以上 | 赤金の背景、専用見出し、強い効果音 |

画面揺れは最大120ms、移動量2pxまでとし、牌姿の読みやすさを損なわない。

## 9. サーバーイベント仕様

現在の`win`イベントには、他家へ公開する和了手牌と盾牌判定の詳細が不足している。次の情報を1つの確定済み結果として送る。

```javascript
{
  type: 'win',
  resultId,
  winType,
  winner,
  loser,
  winningTile,
  concealedHand,
  melds,
  doraIndicators,
  yaku,
  han,
  scoreValue,
  scoresBefore,
  scoreDeltas,
  scoresAfter,
  waits,
  shieldResolution: {
    disabledByOpenRiichi,
    matched: [
      {
        playerIdx,
        shieldTile,
        preventedPayment
      }
    ]
  }
}
```

クライアント側で役や点数を再計算しない。サーバーが確定した同一結果を全員へ送り、全画面で同じ演出を再生する。

ダブロン時は`win`を複数回続けて送らず、1つの`round_result`に`wins`配列として格納し、和了者ごとに牌姿と役を順番に表示する。

## 10. 次局への同期

現在は和了イベント送信後3秒で次局が開始されるため、本仕様の演出時間を確保できない。次局開始条件を次へ変更する。

- サーバーは結果送信後、`ROUND_RESULT`状態で停止する
- 人間プレイヤーは演出完了後に`result_ready`を送信する
- NPCは即時ready扱いとする
- 全人間プレイヤーがready、かつ結果送信から最低6秒経過後に次局へ進む
- 未応答がいても最大20秒で次局へ進む
- 再接続時は同じ`resultId`と結果データを再送する

`次局へ`は自分の演出を完了させるボタンであり、単独でサーバーを即時進行させない。

## 11. 音

音声の種類、音量、再生タイミング、同時発音制御、設定UIは
[盾麻雀 音声演出仕様](./audio-presentation-spec.md)に従う。

和了演出では、ロン・ツモ宣言、牌姿開示、役の逐次表示、盾牌判定、得点確定、点数移動をそれぞれ異なる音で表現する。早送り時は未再生音を連打せず、スキップ音1回へ置き換える。

## 12. 受け入れ条件

- 他家の和了でも、和了者の手牌・副露・和了牌を確認できる
- 役が1件ずつ表示され、途中で早送りできる
- 結果画面を最低6秒は確認できる
- ツモ時、盾牌防御の成立者と防いだ支払い額が分かる
- オープン立直による盾牌防御無効が区別できる
- 通常局では今回の点数移動を和了点と同時に確認できる
- 最終ゲームでは各家の累計得点と順位を確認できる
- ダブロンでも各和了者の牌姿と役を失わない
- 再接続しても同じ結果画面を復元できる
- アニメーション無効設定でも全情報が欠落しない

## 13. 実装順

1. サーバーの和了結果データを拡張する
2. 3秒固定の次局タイマーを結果ready方式へ変更する
3. 牌姿開示パネルを実装する
4. フェーズ制御と役の逐次表示を実装する
5. 盾牌判定演出を実装する
6. 点数移動演出を実装する
7. ダブロン、再接続、低モーション設定を対応する
