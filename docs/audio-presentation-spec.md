# 盾麻雀 音声演出仕様

## 1. 目的

音声は派手さの追加だけではなく、画面を注視していなくても対局状態の変化を把握できる情報伝達として使用する。

優先順位は次の通り。

1. 自分が操作すべき状態を知らせる
2. 打牌・鳴き・立直・盾交換の成立を確認させる
3. 和了と盾牌防御の結果を段階的に理解させる
4. 対局の緊張感と節目を作る

音を消しても情報が欠落しないこと、音を出しても長時間の対局で疲れにくいことを必須とする。

## 2. 基本方針

- 初期実装ではBGMとキャラクターボイスを導入せず、短い効果音を中心にする
- 牌操作音は乾いた木・樹脂系、盾牌音は低い金属または硬質ガラス系で音色を分ける
- 通常操作音は100ms前後、重要通知は300ms前後、和了演出は最大1.2秒程度とする
- 同じ種類の音を連続再生する場合は音量と音程に小さな差を付け、機械的な連打感を抑える
- 他家の通常打牌は小さく、自分の操作と重要イベントは明瞭にする
- 高い音を通知、低い音を確定・失敗・危険の意味に統一する
- ユーザー操作前に勝手に音を再生せず、ブラウザの自動再生制限に従う

## 3. 音量カテゴリ

音量はマスター音量の下に4系統を設ける。

| カテゴリ | 初期値 | 内容 |
| --- | ---: | --- |
| 操作音 | 55% | ツモ、打牌、理牌、ボタン |
| 宣言・通知音 | 75% | 鳴き、立直、手番、残り時間 |
| 盾牌音 | 75% | 盾選択、盾交換、防御成立、無効 |
| 結果音 | 85% | ロン、ツモ、役表示、点数確定 |

マスター音量の初期値は70%とする。ミュート状態と各音量は`localStorage`へ保存する。

## 4. 通常対局の効果音

### 配牌・ツモ

| イベント | 音 | 条件 |
| --- | --- | --- |
| 配牌開始 | 牌を複数枚流す短い音 | 1局につき1回 |
| 自分のツモ | 牌を卓へ置く明瞭な音 | 自分のみ少し大きい |
| 他家のツモ | 原則無音 | 画面上で牌が増える場合のみ極小音 |
| 王牌・ドラ表示 | 小さなめくり音と高いアクセント | ドラ追加時も同じ |

全員のツモに強い音を付けると頻度が高すぎるため、他家の通常ツモは鳴らさない。

### 打牌

- 自分の打牌: 中央定位、短く明瞭な牌音
- 他家の打牌: 席方向に応じて軽く左右へ定位
- ツモ切りと手出しで音色は変えない。視覚表示だけで区別する
- 立直宣言牌: 通常打牌音の直後に立直音を重ねる
- 同時再生上限は2音とし、古い打牌音を停止して音割れを防ぐ

### 理牌・選択

- 自動理牌: 30msから50msの軽い整列音を1回だけ鳴らす
- 牌選択: 原則無音
- 選択解除: 原則無音
- 不正な操作: 低い短音を1回。連打時は500msのクールダウンを設ける

クリックするたびに音を鳴らすと疲労につながるため、選択状態の変化は視覚表現を主とする。

## 5. 操作要求と制限時間

### 自分の手番

- 通常の自分手番開始は、柔らかい高音を1回
- 鳴き・ロン・ツモなど選択が必要な場合は、通常手番と異なる2音の通知
- ブラウザが非表示状態の場合も通知音は鳴らしてよい。ただしOS通知は別設定とする

### カウントダウン

- 残り3秒、2秒、1秒で短いクリック音
- 最後の1秒だけ音程を上げる
- 自動パスまたは自動打牌成立時は低い確定音
- 他家の思考時間カウントダウンは鳴らさない

## 6. 鳴き・槓・立直

| イベント | 音声演出 |
| --- | --- |
| チー | 軽い上昇音。ポンより弱い |
| ポン | 二打の硬い牌音 |
| カン | 低い衝撃音と短い余韻 |
| 立直 | 打牌音の後、棒を置く硬い音と上昇アクセント |
| オープン立直 | 立直音に高い金属音を追加し、通常立直と明確に区別 |
| ロン候補出現 | 緊急性のある短い2音通知 |
| ツモ候補出現 | ロン候補より柔らかい上昇通知 |

鳴きの名称を読み上げるボイスは初期実装に含めない。宣言種別は画面文字で必ず表示する。

## 7. 盾牌の音

盾牌は通常牌と異なる機構であるため、一貫して硬質な専用音色を使用する。

### 盾牌選択

- 手牌から盾牌へ移動: 小さな金属クリック
- 盾牌から手牌へ戻す: 逆方向の軽いクリック
- 選択確定: 低い締まったロック音
- 時間切れ自動確定: ロック音の前に短い警告音

### 盾交換

1. 閉じた盾牌を選択した時: 小さな選択音
2. 手牌をクリックして交換成立: 盾牌を抜く音
3. 手牌が公開盾牌になる瞬間: 硬質な反響音
4. 河へ盾牌が置かれる瞬間: 通常より少し重い打牌音

4音を長く並べず、交換成立時に250msから400msの一連の音としてまとめる。

### 盾牌防御

- 防御成立: 低い衝撃を受け止めた後、澄んだ上昇音
- 複数人防御: 衝撃音は1回、成立者ごとに短い高音を追加
- 防御なし: 原則無音
- オープン立直による無効: 防御成立音の冒頭を鳴らした後、遮断する低い音
- 表向き盾牌の没収: 控えめな消失音

盾牌防御音は本作固有の主要フィードバックなので、通常の鳴き音や打牌音と混同しない専用素材を使う。

## 8. 和了演出の音声タイムライン

[和了演出仕様](./win-presentation-spec.md)の画面フェーズと同期する。

| フェーズ | 音 |
| --- | --- |
| 進行停止 | 卓上音を100msで減衰させ、短い吸い込み音 |
| ロン・ツモ宣言 | 種別ごとの主効果音 |
| 牌姿開示 | 牌を展開する短い連続音 |
| 役表示 | 1役ごとに軽い音。高ハン役は少し低く重くする |
| 盾牌判定 | 防御成立音、無効音、または無音 |
| 合計ハン確定 | カウントアップ後に確定音 |
| 点数移動 | 移動開始音、加点・減点の着地音 |
| 結果保持 | 音を止め、読み取りを妨げない |

### ロン・ツモ

- ロン: 鋭い衝撃と短い下降音。放銃の瞬間性を表す
- ツモ: 上昇音から強い着地音。自力完成を表す
- 槍槓: ロン音に金属を断つ音を追加

ボイスを導入する場合も、効果音を消してボイスだけにせず、両者を別音量カテゴリにする。

### 役の逐次表示

- 各役の表示音は50msから100ms
- ドラ・赤ドラは通常役より明るい音
- 8ハン以上の役または高打点役は低いアクセントを追加
- 役数が7件以上の場合、すべてに強い音を付けず、後半は音量を20%下げる
- 早送り時は未再生の役音を連打せず、短いスキップ音1回に置き換える

### 打点別

| 合計ハン | 主効果音 |
| --- | --- |
| 1-3ハン | 通常の和了音 |
| 4-5ハン | 金属的なアクセントを追加 |
| 6-7ハン | 低音と長めの余韻を追加 |
| 8ハン以上 | 専用の強い確定音。最大1.2秒 |

音量差だけで強さを表現せず、音色と余韻で差を付ける。最大ピーク音量はどの打点でも同じ上限に収める。

## 9. 点数移動

- 加点: 明るい上昇音
- 減点: 短い下降音
- 0点移動: 無音
- 盾牌防御による0点: 専用の盾音を鳴らす
- 複数人の点数移動は同時に開始し、1人ずつ音を重ねない
- 最終点数確定時に1回だけ着地音を鳴らす

点数のカウント中に細かな連続音を鳴らす場合は最大8回とし、長い電子音の連打を避ける。

## 10. 流局・局終了・ゲーム終了

| イベント | 音 |
| --- | --- |
| 流局 | 中立的な下降音 |
| 聴牌公開 | 軽い牌展開音 |
| ノーテン盾没収 | 控えめな盾消失音 |
| 次局開始 | 短い開始音 |
| ゲーム終了 | 1.5秒以内の終局音 |
| 順位表示 | 自分の順位に応じて音色を変えるが、過度な敗北音は使わない |

## 11. BGM

初期リリースではBGMなしを推奨する。理由は、対局時間が長く、効果音の識別性と疲労評価を先に行う必要があるため。

BGMを後から導入する場合:

- 通常対局: ループが目立たない低密度曲
- 終盤または点差拡大時: 自動で曲を切り替えない
- 和了宣言時: 150msで-12dBまでダッキング
- 結果保持時: 500msで元の音量へ戻す
- BGMと効果音は個別にミュート可能

局面に応じた自動BGM変更は、プレイヤーへ危険度などの不必要な暗示を与えるため採用しない。

## 12. ボイス

初期実装では採用しない。導入する場合の対象は以下に限定する。

- ロン
- ツモ
- チー
- ポン
- カン
- 立直
- オープン立直

プレイヤー名、役名、点数は読み上げない。ボイス音量は独立設定とし、効果音だけでイベント種別を判別可能にする。

## 13. 技術仕様

### 再生方式

- Web Audio APIを使用する
- 最初のユーザー操作で`AudioContext.resume()`を実行する
- 短い効果音は起動後または対局開始時にプリロードする
- 対局中に未取得音源を待って進行を止めない
- 読み込み失敗時は無音で続行し、ゲームロジックへ影響させない

### 推奨形式

- 配信形式: OGG VorbisとMP3の2形式
- 原本: WAV、48kHz、24bit
- モノラル: 牌音、通知音、ボタン音
- ステレオ: 和了音、終局音
- 通常効果音は100KB未満、和了音は300KB未満を目安とする

### 同時発音

- 全体の同時発音上限: 8
- 打牌音: 最大2
- UI音: 最大1
- 役表示音: 最大2
- 同一音源の再発音間隔: 40ms以上
- 重要音が鳴る際は、通常操作音を-6dBダッキングする

### イベントAPI

画面コードから直接ファイル名を指定せず、意味イベントを呼ぶ。

```javascript
audio.play('tile.discard', { seat: playerIdx });
audio.play('call.riichi');
audio.play('shield.exchange');
audio.play('shield.defended', { count: 2 });
audio.play('win.ron', { han: 6 });
```

音源パス、音量、ピッチ差、同時発音制御は`audio-manager.js`へ集約する。

## 14. 設定UI

対局画面の邪魔にならない歯車メニューに配置する。

- 全ミュート
- マスター音量
- 操作音
- 宣言・通知音
- 盾牌音
- 結果音
- 将来用: BGM
- 将来用: ボイス
- バックグラウンド時の通知音

ミュートボタンは対局画面から1クリックで操作可能にする。

## 15. アクセシビリティ

- 音だけで情報を伝えず、必ず文字・色・アニメーションを併用する
- 防御成立、時間切れ、操作エラーには短い文字通知も表示する
- 音に敏感な利用者向けに「重要通知のみ」プリセットを設ける
- `prefers-reduced-motion`は音量へ自動連動させない
- 初回起動時に突然大音量を出さない

### 音量プリセット

- 標準: 全カテゴリ有効
- 静音: 操作音20%、通知音50%、盾牌音50%、結果音60%
- 重要通知のみ: 手番、時間切れ、ロン・ツモ、盾牌防御のみ
- 無音: 全カテゴリ0%

## 16. 実装優先順位

### Phase 1: 必須

1. ミュートとマスター音量
2. 自分の手番・残り3秒通知
3. 打牌、鳴き、立直、オープン立直
4. 盾交換、盾牌防御、盾防御無効
5. ロン、ツモ、役表示、点数確定

### Phase 2: 改善

1. 席方向による打牌音の定位
2. 打点別の和了音
3. 流局、終局、順位表示
4. 音量カテゴリ別設定

### Phase 3: 任意

1. BGM
2. 宣言ボイス
3. 音声テーマの切り替え

## 17. 受け入れ条件

- 初回ユーザー操作後にのみ音声再生が開始される
- 自分の手番、残り時間、和了候補を音で区別できる
- チー、ポン、カン、立直、オープン立直を区別できる
- 通常牌と盾牌の操作音を区別できる
- 盾牌防御成立とオープン立直による無効を区別できる
- 役の逐次表示と音が同期し、早送り時に音が連打されない
- 和了結果の点数移動と音が同期する
- ミュート設定が再読み込み後も維持される
- 音源読み込み失敗で対局が停止しない
- 長時間対局で通常打牌音が和了音より目立たない
