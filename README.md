# 盾麻雀

盾麻雀（Shield Mahjong）は、日本式立直麻雀をベースに「盾牌」という防御用の牌を加えた4人対戦Webゲームです。

通常の手牌13枚に加えて、各プレイヤーは局の開始時に伏せた盾牌を持ちます。盾牌は手作りにも守備にも関わり、ツモ和了への支払いを防ぐことがあります。

## まず知っておくこと

- 4人用の立直麻雀ベースです。
- 各プレイヤーは最初に16枚を受け取り、そのうち3枚を盾牌、残り13枚を手牌にします。
- 点数は符なしの翻数制です。
- 1ゲームは最大8局です。
- 全員0点から開始し、マイナス点もあります。
- トップとラスの点差が50,000点を超えるか、8局終了でゲーム終了です。
- 赤ドラは赤7がデフォルトです。部屋設定で赤5に変更できます。

## 盾牌とは

盾牌は各プレイヤーの前に伏せて置かれる特殊な牌です。

局開始時、配られた16枚から盾牌を選びます。盾牌にした牌は手牌には入りません。残り13枚が通常の手牌になります。

### 盾牌の状態

盾牌には2つの状態があります。

- 伏せ盾牌: 自分だけ中身が見える盾牌
- 公開盾牌: 全員に見えている盾牌

局開始時の盾牌は伏せ盾牌です。

## 盾交換

自分の手番では、伏せ盾牌と手牌を交換できます。

交換すると次のようになります。

1. 選んだ伏せ盾牌を河へ捨てます。
2. 選んだ手牌を盾牌エリアへ移します。
3. 盾牌エリアへ移した手牌は公開盾牌になります。

つまり、盾交換は「盾牌を1枚切って、手牌1枚を公開盾にする」行動です。

公開盾牌は後述の盾牌防御に使われます。一方で、手牌の一部を公開して固定するため、守備の代償として手牌構成を失います。

## 盾牌防御

盾牌防御はツモ和了に対してだけ発生します。

誰かがツモ和了したとき、自分の公開盾牌の中に和了者の待ち牌があれば、自分のツモ支払いは0点になります。

例:

- Aが4萬待ちでツモ和了
- Bの公開盾牌に4萬がある
- Bはそのツモ支払いを防御し、支払い0点

複数人が条件を満たす場合は、それぞれ防御します。

### 盾牌防御が効かない場合

次の場合、盾牌防御は効きません。

- ロン和了
- 流し満貫
- 和了者がオープンリーチしているツモ和了

オープンリーチでツモられた場合、公開盾牌に待ち牌があっても防御は無効です。

## 立直とオープンリーチ

立直とオープンリーチはどちらも門前テンパイ時に宣言できます。

- 立直: 1翻
- オープンリーチ: 1翻

オープンリーチでは手牌が公開されます。オープンリーチで和了したツモは、他家の盾牌防御を無効にします。

立直またはオープンリーチの宣言打牌として、盾交換を使うこともできます。

### 立直後の行動

立直後は、通常の立直麻雀と同じく自由な手替わりはできません。

- 和了できる場合はロン・ツモできます。
- ツモれない通常のツモ牌は自動で切られます。
- 待ちが変わらない暗槓だけ可能です。
- 立直宣言牌が鳴かれた場合は、次の自分の捨て牌が横向き表示になります。

## 局の終わりと盾牌の持ち越し

和了または流局で局が終わると、盾牌は次のように処理されます。

### 和了時

- 和了者: すべての盾牌を失います。
- 非和了者: 伏せ盾牌だけ次局へ持ち越します。
- 公開盾牌: 持ち越せません。

### 流局時

- テンパイ者: 伏せ盾牌だけ次局へ持ち越します。
- ノーテン者: すべての盾牌を失います。
- 公開盾牌: 持ち越せません。

持ち越した盾牌がある場合、次局の盾牌選択枚数は3枚より多くなります。

## 点数

盾麻雀では符計算を使いません。翻数だけで点数を決めます。

| 翻数 | 点数 |
| ---: | ---: |
| 1翻 | 1,200点 |
| 2翻 | 2,400点 |
| 3翻 | 4,800点 |
| 4翻 | 9,600点 |
| 5翻 | 12,000点 |
| 6翻以上 | 以後1翻ごとに+2,400点 |

ロンの場合は放銃者が全額を支払います。

ツモの場合は3人で分担します。各支払いは100点単位に丸められます。盾牌防御したプレイヤーは支払い0点になります。

リーチ棒、積み棒、親の加点、符、満貫・跳満・倍満などの段階点はありません。

## 一般的な立直麻雀との主な違い

- 盾牌と盾交換があります。
- ツモ和了に対して盾牌防御が発生します。
- オープンリーチがあります。
- 符計算がありません。
- 親の点数増減、連荘、リーチ棒、積み棒はありません。
- 赤ドラは赤7がデフォルトです。
- 清一色は門前・副露を問わず5翻です。
- 混一色は門前・副露を問わず2翻です。
- 一気通貫、三色同順、混全帯么九、純全帯么九などに喰い下がりはありません。
- 一槓子、二槓子があります。
- 一部の役満は8翻役として扱われます。
- ダブル役満や待ち形による役満増加はありません。

## 基本操作

### 部屋を作る

1. 名前を入力します。
2. 必要なら部屋設定を変更します。
3. 「部屋を作成」を押します。
4. 表示された部屋IDを他のプレイヤーに共有します。

### 部屋に参加する

1. 名前を入力します。
2. 部屋IDを入力します。
3. 「参加」を押します。

ホストは、4人に満たない場合でもNPCを補充して開始できます。

### 盾牌を選ぶ

局開始時に盾牌選択画面が出ます。

- 盾牌にしたい牌をクリックします。
- 盾牌エリアの牌をクリックすると手牌候補へ戻せます。
- 必要枚数を選んだら「確定」を押します。
- 画面にはその局の自風とドラ表示牌が表示されます。

### 打牌・鳴き・和了

手番になると、可能な行動ボタンが表示されます。

- 手牌クリック: 打牌
- ツモ: ツモ和了
- リーチ: 立直宣言
- オープンリーチ: 手牌公開立直
- 暗槓 / 加槓: カン

鳴ける場合は、ロン・ポン・チー・カン・パスが表示されます。

チーの候補が複数ある場合は、組み合わせを選んでからチーします。

### 盾交換する

1. 自分の盾牌エリアから伏せ盾牌をクリックします。
2. 交換したい手牌をクリックします。
3. 選んだ盾牌が河へ出て、選んだ手牌が公開盾牌になります。

リーチまたはオープンリーチを押したあとに伏せ盾牌を選ぶと、盾交換を宣言打牌として使えます。

## 部屋設定

### 立直・オープン立直時に盾牌を全オープン

オンにすると、立直またはオープンリーチ宣言時に自分の伏せ盾牌がすべて公開されます。

デフォルトはオフです。

### 赤ドラ

赤ドラの数字を選べます。

- 赤7: デフォルト
- 赤5: 一般的な赤5に近い設定

## ローカルで遊ぶ

Node.jsが必要です。

```bash
npm install
npm start
```

起動したらブラウザで次を開きます。

```text
http://localhost:3000/
```

開発中に自動再起動したい場合は次を使えます。

```bash
npm run dev
```

npm test

## 技術構成

- Node.js
- Express

主な実装ファイル:

- `src/mahjong/game-engine.js`: ゲーム進行、盾牌、和了、流局

- `src/mahjong/scoring.js`: 点数計算

- `public/js/app.js`: クライアント操作と表示
